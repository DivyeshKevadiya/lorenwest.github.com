Class: Monitor

This is the singleton class loaded on require('monitor'), and is the primary interface between node-monitor and the running application.

It offers probe registration so a UI can discover existing probes and instantiate new ones.

To monitor an application:

var Monitor = require('monitor');
Monitor.start('My Application');

This starts the default probes and allows the UI to attach to this process.

registerProbe

void registerProbe ( probeInstance )
Register a probe instance, and retrieve an ID for the instance. Probes are registered so they can receive an ID to attach to their monitor events. The ID can be back-referenced to the probe class and instance that generated the event. There is no un-register, which has the benefit of retaining information about events for the duration of the applicaiton. The drawback is a reference to the probe is long-held and never garbage collected. Keep this in mind when designing probes.
Parameters:
probeInstance <Probe> - Instance of the probe class being registered.
Returns: void
